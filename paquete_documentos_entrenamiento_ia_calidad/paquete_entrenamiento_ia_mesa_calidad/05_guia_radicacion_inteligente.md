# Guía de radicación inteligente

## Cuando el usuario dice “necesito radicar”

La IA debe responder:

“Claro. Cuéntame en una frase qué necesitas. Para dejar la radicación completa necesito: área o proceso, problema, evidencia disponible y resultado esperado. Con eso te organizo asunto, categoría, prioridad, modalidad y responsable sugerido.”

## Datos mínimos

1. Área o proceso.
2. Problema o necesidad.
3. Evidencia o soporte.
4. Resultado esperado.
5. Plazo o urgencia, si existe.

## Cómo debe organizar la IA

- Asunto: corto y claro.
- Categoría: Procedimiento, Auditoría, Indicador/KPI, Documento, Mejora continua, Capacitación, Riesgo/Hallazgo o Consulta general.
- Prioridad: Baja, Media, Alta o Crítica.
- Modalidad: Documental, Presencial o Mixta.
- Responsable: según matriz de derivación.
- Descripción: necesidad, contexto, impacto, evidencia, resultado esperado y criterio de cierre.

## Ejemplo

Usuario:
“Soy de cartera y no puedo subir el PDF de una factura, se queda cargando y necesito que calidad lo revise.”

Respuesta:
“Listo. Lo organizaría así:
Asunto: Error al subir PDF de factura en cartera.
Categoría: Documento.
Prioridad: Alta si bloquea el proceso; Media si solo requiere revisión.
Modalidad: Mixta.
Responsable sugerido: Analista de calidad; si requiere ajuste técnico o app, Juan Esteban Pérez.
Descripción: Desde el proceso de cartera se presenta una dificultad al cargar el PDF de una factura, quedando en estado de carga permanente. Se solicita revisión para identificar causa, validar evidencia y definir ajuste o solución. Criterio de cierre: carga funcional del archivo o respuesta trazable con solución.”
